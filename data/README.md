# `data/`

Datasets used by the thesis experiments. The folder follows the common
`raw` → `processed` convention: never edit `raw/` by hand; everything in
`processed/` is regenerated by the ETL notebooks from `raw/`.

```text
data/
├── raw/        # original Knapsack Problem (KP) instances — read-only
└── processed/  # features and unified tables produced by the ETL
```

## `raw/` — KP instances

Generated KP instances, grouped into sets named `Set-<n_items>-<capacity>`
(e.g. `Set-25-32` = 25 items, capacity 32). Each set holds 400 `.kp` files.

**`.kp` file format**

```text
<n_items>, <capacity>
<weight>, <profit>
<weight>, <profit>
...
```

The first line gives the number of items and the knapsack capacity; each
remaining line is one item's weight and profit. Filenames encode the
generator and difficulty, e.g. `DEF_EASY_25_000.kp`.

The instances were produced with the evolutionary KP instance generator of
Plata-González et al. (2019):

> Plata-González, L. F., Amaya, I., Ortiz-Bayliss, J. C., Conant-Pablos, S. E.,Terashima-Marín, H., & Coello Coello, C. A. (2019). Evolutionary-based tailoring of synthetic instances for the Knapsack problem. *Soft Computing*,*23*, 12711–12728. <https://doi.org/10.1007/s00500-019-03822-w>

## `processed/` — ETL outputs

Produced by the ETL notebook (`notebooks/01-etl-instances.ipynb`).
Currently under `etl_2026-02-25/`:

- `kp_features.csv` — one row per instance with engineered features (size,
  capacity ratio, weight/profit statistics, weight–profit correlation,
  dominant heuristic, instance type).
- `kp_instances_unified.csv` — all instances flattened to one long table
  (`filename, set, dominant_heuristic, instance_type, weight, profit`).

## Notes

- To regenerate `processed/`, run the ETL notebooks against `raw/`.
- Large files here are tracked/ignored deliberately — see the root `.gitignore`.
